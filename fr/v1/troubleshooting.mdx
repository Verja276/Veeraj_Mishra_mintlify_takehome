# Guide de dépannage

Ce guide vous aide à résoudre les problèmes courants avec Salsa as a Service. Si vous ne trouvez pas de solution ici, notre équipe de support est prête à vous aider.

## Diagnostics rapides

Avant de plonger dans des problèmes spécifiques, effectuez ces vérifications rapides :


1. **Vérifier l'état du système**
   Visitez notre [page d'état](https://status.salsaasaservice.com) pour voir s'il y a des problèmes de service en cours.

2. **Vérifier la connexion Internet**
   Assurez-vous d'avoir une connexion Internet stable et que vous pouvez accéder à d'autres sites web.

3. **Vider le cache du navigateur**
   Videz le cache et les cookies de votre navigateur, puis essayez de vous reconnecter.

4. **Vérifier la compatibilité du navigateur**
   Assurez-vous d'utiliser un navigateur pris en charge (Chrome, Firefox, Safari, Edge).

#### Problèmes de connexion {#problemes-connexion}

**Problème** : Impossible de se connecter à votre compte

**Solutions :**


1. **Vérifier les identifiants**
   Vérifiez votre adresse e-mail et mot de passe. Assurez-vous que Caps Lock est désactivé.

2. **Réinitialiser le mot de passe**
   Utilisez le lien "Mot de passe oublié" sur la page de connexion pour réinitialiser votre mot de passe.

3. **Vérifier l'e-mail**
   Recherchez les e-mails de vérification de compte dans votre dossier spam.

4. **Contacter le support**
   Si le problème persiste, contactez le support avec votre adresse e-mail de compte.



> **Warning:**
**Note de sécurité** : Ne partagez jamais votre mot de passe avec quiconque. Notre équipe de support ne vous demandera jamais votre mot de passe.


### Synchronisation des données

#### Problèmes d'intégration POS

**Problème** : Les données POS ne se synchronisent pas avec Salsa as a Service


#### Square

1. **Vérifier la connexion Square**
   Vérifiez que l'intégration Square est toujours connectée dans votre tableau de bord.

2. **Réautoriser l'intégration**
   Déconnectez et reconnectez votre compte Square.

3. **Vérifier les permissions API**
   Assurez-vous que les permissions de l'API Square incluent la gestion d'inventaire.

#### Toast

1. **Vérifier la clé API Toast**
   Vérifiez que vos identifiants API Toast sont valides et non expirés.

2. **Vérifier le mappage du menu**
   Assurez-vous que les articles du menu salsa sont correctement mappés aux articles d'inventaire.

#### Général

1. **Vérifier la connexion réseau**
   Assurez une connexion Internet stable pour les deux systèmes.

2. **Consulter les journaux de synchronisation**
   Vérifiez les journaux d'intégration pour les messages d'erreur.

3. **Tester la synchronisation manuelle**
   Essayez de déclencher manuellement une synchronisation de données depuis votre tableau de bord.



### Problèmes de production

#### Problèmes de contrôle qualité

**Problème** : Les scores de qualité ne s'enregistrent pas correctement

**Solutions :**

1. **Vérifier la calibration des appareils** : Assurez-vous que les appareils de test de qualité sont correctement calibrés
2. **Revoir les procédures de test** : Vérifiez que le personnel suit les protocoles de test corrects
3. **Mettre à jour le logiciel** : Assurez-vous que l'application mobile est mise à jour vers la dernière version
4. **Vérifier les permissions** : Vérifiez que le personnel a les permissions appropriées pour les tests de qualité

### Problèmes d'application mobile

#### Plantages d'application

**Problème** : L'application mobile plante ou se fige

**Solutions :**


1. **Mettre à jour l'application**
   Assurez-vous d'exécuter la dernière version de l'application mobile Salsa as a Service.

2. **Redémarrer l'application**
   Forcez la fermeture de l'application et redémarrez-la.

3. **Vider les données de l'application**
   Videz le cache et les données de l'application (vous devrez vous reconnecter).

4. **Réinstaller l'application**
   Désinstallez et réinstallez l'application depuis l'app store de votre appareil.
